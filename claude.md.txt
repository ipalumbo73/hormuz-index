Logica di deduplica
Obiettivo
Clusterizzare articoli diversi che parlano dello stesso evento.

Strategia
fingerprint su:

titolo normalizzato,

attori,

location,

categoria,

bucket temporale,

URL canonicale.

similarità titolo/summary con RapidFuzz.

cluster se similarità > 88 e attori/location coerenti.

se GDELT event_id è disponibile, usarlo come strong hint ma non come unica chiave.
​

Regole
Non deduplicare eventi distinti accaduti nello stesso luogo a ore diverse.

Preferire come canonical article la fonte con reliability più alta.

Conservare tutte le fonti nel cluster.

Classificazione segnali
Approccio
Fase 1 rule-based robusta.

Fase 2 opzionale LLM per migliorare precisione.

Fase 3 override manuale admin.

def pipeline_cycle():
    raw_items = collect_from_all_sources()
    normalized_articles = normalize_articles(raw_items)
    stored_articles = persist_articles(normalized_articles)

    candidate_events = extract_events(stored_articles)
    clustered_events = cluster_events(candidate_events)
    classified_events = classify_and_score(clustered_events)

    persist_events(classified_events)

    indices = recompute_indices(windows=["24h", "7d", "30d"])
    noi = compute_noi(indices["noi_components"])
    indices["NOI"] = noi

    scenarios = compute_scenarios(indices)
    explanations = build_explanations(indices, scenarios, classified_events)

    persist_index_snapshot(indices)
    persist_scenario_snapshot(scenarios, explanations)

    alerts = evaluate_alert_rules(indices, scenarios)
    dispatch_alerts(alerts)

# Risk Model

## Nuclear Opacity Index (NOI)
NOI measures the loss of international verifiability over the Iranian nuclear program.

### Components
- A site_access_loss = 25%
- B material_knowledge_loss = 25%
- C enrichment_verification_gap = 20%
- D underground_activity_signal = 10%
- E technical_diplomatic_breakdown = 10%
- F conflicting_narratives_uncertainty = 10%

### Formula
NOI = 0.25A + 0.25B + 0.20C + 0.10D + 0.10E + 0.10F

Clamp result to 0-100.

### Hard rules
- if A >= 75 and B >= 90 => NOI minimum 80
- if C >= 75 and D >= 50 => NOI += 5
- if E >= 80 and F >= 70 => NOI += 3
- clamp final result to 0-100

### Thresholds
- 0-24 green
- 25-49 yellow
- 50-69 orange
- 70-84 red
- 85-100 dark red

# Risk Model

## Indices
- NOI
- GAI
- HDI
- PAI
- SRI
- BSI
- DCI

## Nuclear Opacity Index (NOI)
Purpose:
Measure loss of international verifiability regarding the Iranian nuclear program.

Range:
0-100

### NOI components
- A: site_access_loss = 25%
- B: material_knowledge_loss = 25%
- C: enrichment_verification_gap = 20%
- D: underground_activity_signal = 10%
- E: technical_diplomatic_breakdown = 10%
- F: conflicting_narratives_uncertainty = 10%

### Formula
NOI = 0.25*A + 0.25*B + 0.20*C + 0.10*D + 0.10*E + 0.10*F

Clamp result 0..100.

### Hard rules
- if A >= 75 and B >= 90 then NOI = max(NOI, 80)
- if C >= 75 and D >= 50 then NOI += 5
- if E >= 80 and F >= 70 then NOI += 3
- clamp final NOI 0..100

### NOI thresholds
- 0-24 green
- 25-49 yellow
- 50-69 orange
- 70-84 red
- 85-100 dark red

## Other indices
### GAI
Attacks on Gulf infrastructure, ports, bases, airports, oil/gas assets, cities

### HDI
Shipping disruption, Strait of Hormuz threats, mines, interdiction, boarding, tanker attacks

### PAI
Proxy activity in Lebanon, Yemen, Iraq, Syria and allied networks

### SRI
Strategic rhetoric, existential threat language, “all options”, final window, red lines

### BSI
Signals consistent with nuclear breakout risk or accelerated concealment

### DCI
Diplomatic cooling, Oman talks, GCC mediation, hotlines, reduced raids

## Scenario priors
- contained_conflict = 40
- regional_war = 25
- nuclear_threshold_crisis = 20
- coercive_go_nuclear = 10
- actual_nuclear_use = 5

## Scenario weight matrix
| Index | contained | regional | threshold | coercive | actual |
|---|---:|---:|---:|---:|---:|
| NOI | -0.20 | +0.08 | +0.30 | +0.18 | +0.05 |
| GAI | -0.10 | +0.35 | +0.05 | +0.04 | +0.02 |
| HDI | -0.12 | +0.30 | +0.08 | +0.05 | +0.03 |
| PAI | -0.08 | +0.25 | +0.03 | +0.02 | +0.01 |
| SRI | -0.10 | +0.10 | +0.18 | +0.30 | +0.10 |
| BSI | -0.15 | +0.05 | +0.35 | +0.25 | +0.12 |
| DCI | +0.30 | -0.20 | -0.25 | -0.20 | -0.10 |

## Scenario boost rules
- NOI >= 70 and BSI >= 60 => threshold +10
- NOI >= 80 and SRI >= 65 => coercive +8
- NOI >= 85 and BSI >= 75 and SRI >= 75 => actual +7
- DCI >= 60 => reduce extreme scenarios by 15%

## Time windows
Use:
- 24h weight 0.50
- 7d weight 0.30
- 30d weight 0.20

### Event impact
event_impact = source_reliability * confidence * severity * novelty

### Subindex formula
subindex = sum(event_impact * signal_value) / sum(event_impact)

raw item -> normalized article -> article storage -> event extraction -> clustering -> classification -> signal payload

- GDELT every 15m
- NewsData every 5-10m
- GNews every 10m
- RSS every 5m
- index recompute every 10m

- normalized title
- actor tags
- location tags
- category
- time bucket
- URL canonicalization
- rapidfuzz similarity

## Clustering logic
Cluster related articles into event clusters.

- retries with exponential backoff
- quarantine malformed payloads
- source timeout
- logging per source
- per-source rate limit awareness
